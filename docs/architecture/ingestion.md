# Event Ingestion Enrichment

Detailed reference for auto-property enrichment of incoming events (`internal/geo`, `internal/useragent`, `internal/autoprop`). Linked from the root [`CLAUDE.md`](../../CLAUDE.md) — read this when working on event enrichment.

Incoming events are enriched with auto-properties before being published to NATS:

- **`internal/geo/`** — resolves geo properties (`$country`, `$city`, `$region`, etc.) from proxy-injected HTTP headers. `geo.Provider` is an interface; the Cloudflare implementation reads from `CF-*` headers. Geo properties are **always overwritten** (CDN-injected values are trusted). The visitor IP is **personal data and is never persisted**: the Cloudflare provider resolves geo from `CF-*` headers and never reads the IP, so it is omitted from the `Location`. `geo.ClientIP` exists as the transient extraction primitive a future IP-lookup provider (e.g. MaxMind) would use, but is uncalled today and providers must not emit the IP in the `Location`. `enrichGeo` strips any client-supplied `$ip` from every event (and increments `events.ip_stripped_total`) — so the raw IP never reaches NATS or ClickHouse.
- **`internal/useragent/`** — parses the `User-Agent` header with `ua-parser/uap-go`, normalizing the result to match the web SDK's `navigator.userAgentData` shape: browser/OS names (`Chrome` → `Google Chrome`, `Mobile Safari` → `Safari`, `Mac OS X` → `macOS`, Linux distributions → `Linux`), major-only `$browserVersion` (matching the SDK's low-entropy brand version), multi-part `$osVersion` (matching the SDK's high-entropy `platformVersion`), and an inferred `$mobile` (`"true"`/`"false"`; tablets such as iPad report `false`, matching `navigator.userAgentData.mobile`). Populates `$browser`, `$browserVersion`, `$os`, `$osVersion`, `$device`, and `$mobile`. This is the **server-side fallback** for browsers that do not expose `navigator.userAgentData` — Firefox, Safari, and all iOS browsers — which send no Client Hints; for Chromium browsers the web SDK reports these properties directly (including high-entropy `$osVersion`/`$device` via `navigator.userAgentData.getHighEntropyValues()`). Properties are only written if not already present in `event.AutoProperties`, so SDK-supplied values win. `$device` is only set when a specific device is identified; desktop browsers typically have no `$device`. The parser is initialized once at startup via `useragent.NewParser()`.
- **bot management enrichment** — reads Cloudflare Bot Management headers (injected via Transform Rule) and sets typed auto-properties: `$bot_score` from `CF-Bot-Score` is parsed as `Int64` (0–255, lower = more bot-like) and `$verified_bot` from `CF-Verified-Bot` is parsed as `Bool` (identifies known good bots like Googlebot). Both are **always overwritten** by the server; client-supplied values are stripped before enrichment. Routing through `internal/autoprop` keeps the typed-Variant story consistent across enrichers (geo `$latitude`/`$longitude` are also stored as `Float64`).
- **`internal/attribution/`** — web-analytics derivation (`enrichAttribution`, last in the chain; header-independent, per-event). A pure, stdlib-only `attribution.Derive` computes `$pathname`/`$hostname` from `$url` (parseable http(s) URLs with a host only — no fabricated `/` for garbage), `$referrerDomain` from `$referrer` (any scheme with a host; lowercased, one leading `www.` stripped, **blanked on self-referral**), `$channel` via the normative rule table in `internal/attribution/channel.go` (only when a valid `$url` is present — non-web events carry no channel, not a misleading `Direct`), `$screenSize` from `$screenWidth`×`$screenHeight`, UTM ×5 completion from the `$url` query string (client-sent values win), and `$locale` casing normalization (`en-us` → `en-US`; rollup rows are permanent, so variants must never fragment storage). Per-key overwrite policy: `$referrerDomain`/`$channel` are **server-only** (client values stripped first, mirroring the bot enrichers); `$pathname`/`$hostname`/`$screenSize` derive-if-absent (an SDK may send logical route paths); UTM only-if-absent; `$referrer`/`$pageTitle` pass through as sent. The demo seeder routes through the same `Derive` (`internal/app/seed/clickhouse/attribution.go`), and migration 008's one-shot mutation mirrors it for historical rows (pinned by `TestIntegrationWebAnalytics/mutation_008_matches_attribution_derive`). Derivation is pure and cannot error, but it is not infallible: a `$url` that fails to parse as an http(s) URL with a host derives nothing, which the enricher records as `events.attribution_derive_degraded_total{reason=url_not_web}` so a broken SDK/tenant is visible rather than silent.
